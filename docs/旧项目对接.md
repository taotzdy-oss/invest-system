## 字段含义

| 字段 | 类型 | 用途 | 哪个模块用 |
| --- | --- | --- | --- |
| `legacy_root` | 绝对路径 | 旧项目根 | 全部 |
| `knowledge_base_dir` | 相对路径 | 知识库根 | 知识库 |
| `kb_index_md` / `kb_main_md` / `kb_manifest_json` | 相对路径 | 知识库元信息文件 | 知识库 |
| `review_*` | 相对路径 | 复盘相关 | 复盘 |
| `ladder_*` | 相对路径 | A 股连板天梯（脚本依赖） | 选股 |
| `rebuild_buckets_script` | 相对路径 | 重建分桶脚本 | 复盘 |
| `stock_pick_pool_glob` | 通配 | 观察池目录模式 | 选股 |
| `stock_pick_script_glob` | 通配 | 选股脚本模式 | 策略 / 选股 |

## 适配器对照

| Python 文件 | 暴露函数 | 读取的旧项目对象 |
| --- | --- | --- |
| `adapters/knowledge_base.py` | `list_docs()`, `read_doc()`, `search()`, `manifest_info()` | `锋芒策略解析知识库_*/` 下所有 md/html/csv/json |
| `adapters/review.py` | `list_review_days()`, `ledger_rows()`, `bucket_rows()`, `experience_md()`, `strategy_iteration_md()`, `review_template_md()`, `ledger_index()` | `锋芒爆点_复盘迭代/*` |
| `adapters/stock_pick.py` | `list_pools()`, `get_pool()`, `parse_csv_codes()`, `parse_html_candidates()`, `list_scripts()` | `锋芒爆点_*_打板观察池/`、`generate_fengmang_breakout_*.py` |
| `adapters/strategy.py` | `list_strategy_scripts()`, `get_script()` | 同上选股脚本 |
| `adapters/runner.py` | `run_python()`, `list_recent_logs()` | 任意 python 脚本（子进程） |

## 自定义新策略 / 新数据源

例如未来加一个 `锋芒波段_*` 系列：

1. 在 `config.json -> legacy` 加：
   ```json
   "bo_duan_pool_glob": "锋芒波段_*_周线池",
   "bo_duan_script_glob": "generate_fengmang_boduan_*.py"
   ```
2. 在 `adapters/` 下复制 `stock_pick.py` 改一份 `bo_duan.py`，把目录命名规则换掉。
3. 在 `modules/` 下复制 `picks.py` 改一份 `bo_duan.py`，注册 `/boduan` 路由。
4. 在 `modules/__init__.py` 加 `from modules import bo_duan` 让路由生效。
5. 把入口按钮加到 `templates/layout.html` 或 `modules/home.py`。

> **绝不修改旧脚本**：所有新需求通过新适配器/新模块实现，旧脚本只通过 `adapters/runner.run_python()` 调用。

## 我们触碰旧项目的全部场景

唯一会"写"旧项目目录的操作只有一个：
- **复盘页 → 🔄 重建分层池** 按钮：调用旧 `rebuild_promotion_buckets.py`，该脚本会重写 `锋芒爆点_复盘迭代/数据/晋级分层/*.csv`。

其它一切（笔记、版本、批注、回测）都落到 `data/system.db`，旧项目目录保持原样。
