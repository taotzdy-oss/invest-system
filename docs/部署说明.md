# 部署说明

## 1. 环境要求

| 项 | 要求 |
| --- | --- |
| 操作系统 | macOS / Linux （本系统在 macOS 14+ 验证） |
| Python | 3.9+（macOS 自带 `python3` 即可） |
| 磁盘 | 30 MB 起步（含 sqlite 数据库） |
| 端口 | 默认 8787（仅本机访问） |
| 网络 | 不需要联网；除非你点"运行选股脚本"——旧脚本会去东方财富等接口拉数据 |

确认环境：

```bash
python3 --version          # 期望 >= 3.9.6
which git                  # 期望 /usr/bin/git 或同等
ls /Users/gegezi/Desktop/投资   # 期望旧项目存在
```

## 2. 首次启动

```bash
cd /Users/gegezi/Desktop/投资管理系统
bash scripts/run.sh
```

启动后会自动打开浏览器进入 [http://127.0.0.1:8787](http://127.0.0.1:8787)。
首屏即可看到：
- 最新观察池日期
- 最新复盘日期
- 复盘总样本 / 触板率 / 封住率
- 知识库文档数

## 3. 自检

```bash
bash scripts/check.sh
```

- `tests/test_units.py`：核心组件单测（Markdown 渲染 + 适配器导入 + 旧项目数据可读性）。
- `tests/smoke.py`：起一个临时 server（端口 9911），对 23 条关键路由做请求 + 写操作往返。

如果某项 FAIL，先查 `config.json -> legacy_root` 是否指向了正确的旧项目目录。

## 4. 启动参数

```bash
python3 app.py                 # 默认端口
python3 app.py --port 9000     # 自定义端口
python3 app.py --no-browser    # 不自动开浏览器（如远程 SSH）
PORT=9000 bash scripts/run.sh  # 用环境变量
```

## 5. 数据落地

- 系统**只往** `data/system.db` 写数据。
- 该文件已被 `.gitignore`，不会被提交到 GitHub。
- 想跨机器迁移？把 `data/system.db` 拷贝过去即可，schema 通过启动时 `init_db()` 自动建表/兼容。

## 6. 常见问题

### Q: 端口被占用
```
[错误] 端口启动失败：[Errno 48] Address already in use
```
解决：`PORT=8788 bash scripts/run.sh`，或先 `lsof -i:8787` 找到占用进程。

### Q: 首屏提示"旧项目根目录不存在"
打开 `config.json`，把 `legacy_root` 改成正确路径。

### Q: 知识库搜索没有结果
- 检查 `legacy.knowledge_base_dir` 是否正确（默认是 `锋芒策略解析知识库_20260606`）。
- 如果旧项目知识库目录名带了新日期（例如 `_20260801`），需要更新 `config.json`。

### Q: 调用"运行脚本"按钮卡住
旧 `generate_fengmang_breakout_*.py` 会请求 push2ex.eastmoney.com 等接口，
没联网会 timeout。系统设置了 600s 超时，超时后自动失败并落 `run_logs`。

### Q: 想自定义首页
- 首页拼装在 [modules/home.py](../modules/home.py)
- 样式在 [static/css/app.css](../static/css/app.css)
