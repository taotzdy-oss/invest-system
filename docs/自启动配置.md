# 系统自启动配置

> 由于 macOS 隐私沙盒（TCC）默认不允许 launchd Agent 访问 `~/Desktop`，需要在以下三种方案中选一种。**最省事的是方案 A**。

---

## 方案 A：双击 `.command` 启动 (推荐)

桌面 `投资管理系统/启动投资管理系统.command` 是一个可执行脚本：

1. 在 Finder 双击它（首次会提示"开发者未验证"，按住 Control 点击 → 打开 → 打开）
2. 它会自动启动后台 server + 调度器 + 打开浏览器到 [http://127.0.0.1:8787](http://127.0.0.1:8787)
3. 关闭 Terminal 窗口 = 关闭系统

**开机自启**：把这个 `.command` 加入 **System Settings → General → Login Items**。

## 方案 B：给 Python 3 加 Full Disk Access（一次性，永久自启）

如果想要"开机自动启动 + 进程崩了自动重启"，可以让 launchd 接管：

1. **System Settings → Privacy & Security → Full Disk Access**
2. 点 `+`，输入 `/usr/bin/python3`，添加
3. 同样给 `/bin/zsh` 加 Full Disk Access
4. 然后执行：
   ```bash
   bash /Users/gegezi/invest-system/scripts/launchd/install.sh
   ```
5. 验证：
   ```bash
   launchctl list | grep invest    # 期望非 "-" PID
   curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8787/   # 200
   ```

卸载：`bash /Users/gegezi/invest-system/scripts/launchd/uninstall.sh`

## 方案 C：在 Terminal 里前台启动

```bash
cd /Users/gegezi/invest-system
python3 app.py
# Ctrl+C 退出
```

适合调试 / 临时使用。

## 自动调度任务

无论用哪种启动方式，**只要 server 在运行，后台调度器就在运行**：

| 任务 | 触发 | 行为 |
|---|---|---|
| `ladder_refresh` | 工作日 15:30 / 17:00 | 重导入连板天梯数据，新鲜度检查 |
| `review_daily`   | 工作日 16:00 起 | 检测当日复盘报告是否生成 → 已生成则 sync DB；未生成则标 `report_missing` |
| `picks_daily`    | 工作日 18:00 起 | 检测下一交易日选股脚本 → 存在则跑、回填 plan；缺失则标 `script_missing` |
| `backfill_sync`  | 每小时 | 把旧项目目录的新增数据同步进 DB |
| `health_check`   | 每 10 分钟 | 数据新鲜度 + CSV 校验 + 一致性检查 |

> 关键提醒：`review_daily` 和 `picks_daily` 当前不会**自动生成**复盘报告或选股脚本（它们需要 Codex skill / 你的复盘 agent 介入产生原始文件）。本系统会**正确感知缺失**并显示阻塞原因，不伪造结果。
